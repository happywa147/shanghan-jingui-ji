# 多版本数据模型

## 主要实体

- `work`：著作，如《伤寒论》《金匮要略》。
- `edition`：具体版本或底本。
- `source`：影印本、整理本或数据文件的来源及授权信息。
- `text_unit`：版本自身的最小文本单元，不强制等同于通行条号。
- `alignment`：跨版本对应关系。
- `variant`：字词、句读、增删、次序和方药差异。
- `interpretation`：人工或注家的解释。
- `evidence_claim`：带出处和置信说明的判断。

## 对应关系

`alignment` 必须支持：

- 一对一
- 一对多
- 多对一
- 近似对应
- 无对应
- 尚未判定

系统不得仅以文本相似度自动确认对应关系。算法可以提出候选，但必须保留算法版本、得分和人工审核状态。

## 差异分类

- `orthography`：异体字、通假字或书写差异。
- `punctuation`：句读差异。
- `wording`：用词不同。
- `addition` / `omission`：增文或缺文。
- `split_merge`：条文拆分或合并。
- `ordering`：篇次或条次不同。
- `formula`：方名、药味、剂量、炮制或煎服法不同。
- `structure`：正文、注文、嵌注或缩进层级不同。

## 医理影响

文字差异本身不自动等于医理差异。经人工审核后，可另行标记其是否影响：

- 病位
- 病性
- 病势与传变
- 治法
- 方剂选择
- 药物剂量或煎服
- 禁忌

无法确定时必须保存为 `uncertain`，不得强行归类。
