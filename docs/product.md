# 产品定义

## 核心问题

现有经方资料往往把不同版本、后世注解和整理者判断混为一体。伤寒金匮集首期定位为公开只读文献校勘候选站，解决“依据哪一版本、哪一条来源整理文本、经过谁的判断”这一问题。

## 使用者

首期面向经方研究者和中医师。患者端不在第一阶段范围内。

## 核心流程

```text
原始影印或可靠文本
→ 版本登记
→ 篇章与文本单元切分
→ 跨版本对应
→ 文字差异标注
→ 医理影响审核
→ 方证关系
→ 医案证据匹配
```

## 明确不做

- 不以症状关键词直接匹配处方。
- 不把模型生成内容冒充经典原文或注家原话。
- 不在缺少医生审核时输出可执行处方。
- 不为了统一条号而抹去版本原有篇次和层级。

## 数据层级

1. **原始层**：保留来源文本，不做静默修订。
2. **整理层**：断句、繁简、异体字和结构化字段。
3. **校勘层**：记录版本对应和文字差异。
4. **解释层**：注家意见、人工判断和医理影响。
5. **推断层**：AI 生成的候选辨证与证据链。

不同层级必须分别存储和展示。

## 当前质量状态

- 《伤寒论》三版本文本、对照关系、方剂及条方关系：`imported`。
- 跨版本字符差异：`machine_candidate`。
- 人工校勘结果：由校勘工作台导出，不覆盖导入数据。
- 《金匮要略》四部丛刊俞桥刊本影像：已登记并完成哈希、尺寸和页数校验。
- 《金匮要略》OCR：只允许进入 `machine_ocr`，完成逐页核对前不得称为校定文本。

## 工程基线

执行 `npm run check` 必须同时通过单元测试、来源校验、导入完整性检查、异文生成和报告生成。
