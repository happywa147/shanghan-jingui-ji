# 50条黄金候选 · 五位AI专家预审报告

日期：2026-07-18

## 审查性质

本轮由五位独立 AI agent 从医疗内容安全、版本文献学、数据与算法、中文出版、证据验收五个角度审查 `data/review/golden-candidates.json`。这是机器辅助预审，不是持证中医师或真人文献专家签核；未写入 `first_review`、`second_review` 或 `adjudication` 字段。

## 共同结论

- 50 条当前均为 `awaiting_first_review / candidate`，0 条具备黄金样本资格。
- 约 25 条存在截断、超集、重复合并、相邻条文混入或正文/注文边界污染，必须先重切分。
- 第 31–40 条逐字相同却标为 `approximate`，应检查关系类型语义；它们适合作为真人复核的优先对照组。
- 康平本中的裸“注”“注经”、缺字 `□`、疑似 OCR 字及扩展汉字必须回到影像核验。
- 候选缺少卷、篇、页、行和影像定位，机器置信度不能替代版本证据。
- 古籍病名、方剂、药物和治法只能用于文献对读，不得包装为现代诊疗建议。

## 五个审查维度

1. 医疗内容安全：识别处方、发汗、攻下、利水、麻黄、附子、甘遂等高风险内容，以及急重症和古今术语误读风险。
2. 版本文献学：建议 20 条补定位后优先复审、17 条重建对应、13 条先恢复经注层级或核缺字；当前直接晋级为 0。
3. 数据与算法：发现人审关系类型未写回、审核结构未在晋级路径强校验、分层样本受顺序占坑和重复源单元影响。
4. 中文出版：建议 18 条可保留对齐、3 条需展示分段、4 条需核异文、25 条必须重切；建立底本原貌与现代标点双层。
5. 证据验收：50 条均可回链现有 alignment 与 variant，但注册表为空、缺真人证据；黄金集数量为 0 时不得声称已经建成。

## 已完成的门禁修复

- 审核必须包含合法决定、关系类型、非空证据引用和有效审核时间。
- 双审或裁决确认的关系类型写入 `reviewed_relation_type`，发布时重新计算并核对。
- 增加黄金候选包根 Schema、生成器和算法版本。
- 正式发布策略要求至少 1 条真实黄金样本，避免“空集通过”被误解为人审完成。

## 真人复核顺序

1. 先处理第 31–45、49、50 条等边界较稳或差异较小的样本。
2. 再处理 merge 组及高字符差异组，重点核对条文边界。
3. 第 18、22–26、46–49 条先修复经注结构和缺字，再送双审。
4. 每条由两名不同、已验证、无利益冲突的真人审核；分歧时由第三名独立裁决。

## 统一安全提示

以下内容为古籍不同版本的文献对读，仅用于古籍整理与学术研究，不构成诊断、处方或用药建议。古方中的病名、药名、剂量、炮制与现代临床并非可直接对应，请勿自行尝试；出现呼吸困难、呕血或咯血、抽搐、意识异常等情况应及时就医或联系急救。
