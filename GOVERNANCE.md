# 治理章程

项目当前由 GitHub 仓库所有者代行维护，尚未形成符合95+目标的多人学术治理。任何文档不得虚构专家、机构或审核背书。

## 角色

- 维护者：工程、发布、安全与恢复。
- 版本学审核者：底本、转写、校勘记和对应关系。
- 中药/临床药学安全审核者：药物风险标注，不输出个体处方。
- 独立裁决者：处理双审分歧，不得同时是该记录的初审或复审。

## 决策与回避

- 数据Schema、底本更换、安全政策和发布门禁属重大变更，需至少两名有效维护/审核主体批准。
- 审核者对所在机构、资助、商业利益和亲自编辑的记录声明利益冲突并回避。
- 紧急医疗安全或版权问题可先临时下架，随后必须公开事件、证据和处置。
- 所有撤回和更正只能追加事件，不得静默覆盖。

## 95+治理晋级条件

至少5名真实专家、3个独立背景、bus factor≥3，并连续两个季度公开审核覆盖率、一致率、纠错时效和利益冲突记录。
